## By page type

### Character Profile
You already have **`templates/biblical-character-template-v5-9.html`**. Default sections:
- Reading Stamp (top of page — reading time + complexity dots + TL;DR)
- Theology Card (every body section)
- Timeline Items (Narrative Journey)
- Mini-Card Grid (Themes — 6 cards, two rows of 3)
- Chiastic Structure (when applicable, optional)
- Word Study Card (Hebrew/Greek)
- Scripture Quote
- Key Insight Box
- Bibliography Header (Enhanced) with Cite-this-Study button

**Optional add-ons** worth pulling in for major characters (Abraham, Moses, David, Paul):
- Multi-page navigation (Page Navigation Grid)
- Interactive Hebrew Tooltips through prose
- Eden-Pattern Flip Cards if there's a 4-stage character arc
- Atlas Node cards for "intertextual moments"

**Mobile tabs are auto-generated** from a per-page `<script id="page-section-config" type="application/json">` block in the page's `<head>`. This is a v5.9 architectural change — the previous v5.8.1 system used a 200-entry global config that every page had to inherit. Each page now declares its own section list (id, label, icon, priority) inline, which `character-page-v5-9.js` reads to build the bottom tab bar. See the v5.9 character template for the canonical block.

### Thematic Study (Multi-Part Series)
Pattern proven on Resurrection-as-Revelation, Genesis Double Revelation:

```
HUB PAGE
  HERO — Cosmic Halo or Braided Rope
  ↓ Page Navigation Grid (links to all parts)
  ↓ Theology Card per part with summary
  ↓ Card Fan for "five lenses" if applicable

EACH PART PAGE
  HERO — Cosmic Halo (with Hebrew/Greek Hero Run)
  ↓ Mini-TOC Pill Bar
  ↓ Quick Nav Sidebar Dots
  ↓ Theology Card sections
  ↓ Atlas Node cards for paired comparisons
  ↓ Trajectory Flowchart for the part's argument
  ↓ Critical Distinction Box (1–2 per part)
  ↓ Page Navigation (← prev / next →)
  ↓ Bibliography Header (Enhanced)

CONCLUSION/INTERACTIVE PAGE (optional)
  Iceberg, Venn, or Concentric Rings as the centerpiece
```

### Tool / Workbench Page
Pattern proven on Sermon Builder, Practice Lab, Framework — when the page is something the reader *uses* rather than reads:

```
HERO — Cosmic Spectrum Hero (the signature for tool pages)
  ↓
BREADCRUMBS + skip-link
  ↓
EITHER:
  Multi-Step Wizard (for "produce something" tools — Sermon Builder)
   - Step 1: Nested Accordion Browser (pick the source/preset)
   - Step 2: Selectable Chip System (commit a configuration)
   - Step 3: Form fields with Required Pills + Validation Message Box
   - Step 4: Output Tab Bar with Output Code Blocks per format
OR:
  Section-stack (for instructional tools — Practice Lab, Framework)
   - Theology Cards interleaved with Star Dividers (✦)
   - Worked Example Cards if pedagogy is example-driven
   - Animated Readiness Checklist before any practice action
   - Copy Box for any team-prep / template scaffolds
  ↓
PATHWAY NAVIGATION — prev/next + step indicator (if part of a curriculum)
  ↓
BACK TO TOP FAB — standard on long pages
```

### Multi-Page Character Study (Color-Coded)

For seminary-level character studies that span 5+ interconnected pages — the kind where the reader needs cross-page orientation and a clear sense of progression. Pattern observed across the Noah 8-page suite (hub + corruption + election + ark + flood + covenant + babel + intertext) and applicable to any large character or theme study.

```
HUB PAGE
  Rainbow Covenant Hero — establishes the color spectrum the children will index into
    ↓
  Hub Overview (Theology Card) — Hebrew, etymology, scripture range, study structure
    ↓
  Why X Matters (Significance Panels grid)
    ↓
  Pattern Mini-Map (4-cell preview of the journey arc)
    ↓
  Pattern Section / Cycle visualization — the recurring movements the study will trace
    ↓
  STUDY PAGES (Page Card Grid) — one card per child page, each card's accent matches that page's --page-accent (red/orange/yellow/green/blue/indigo/violet/deep-purple)
    ↓
  Color Guide callout — explains the wayfinding system
    ↓
  Quick Reference (4-card grid: text · themes · motifs · bridge forward)
    ↓
  Visual Timeline / Chronology — if the study has a temporal structure
    ↓
  Methodology Card grid — how the study was made
    ↓
  Academic Note Box — hermeneutical posture
    ↓
  Source Foundations link

CHILD PAGE (one of 7, e.g. noah-flood.html)
  Each child page sets its own --page-accent CSS variable (this is the system's load-bearing trick)
    ↓
  Breadcrumb to hub
    ↓
  Page header with "Page N of M" kicker
    ↓
  Rainbow Stage Tracker — current page's stage gets the pulse
    ↓
  Body sections — Theology Cards consume --page-accent automatically
    Diagrams (Pattern Chain, Cosmos Diagram, etc.) for the page's specific topic
    Callouts (Wordplay Box, Pivot Box, Babel Scene) where appropriate
    Cross-Reference boxes pointing to other pages in the suite
    ↓
  Covenant Thread / theme tracker (if the study has a continuous theme thread)
    ↓
  Key Verses block
    ↓
  Page Navigation — prev / page indicator / next
    ↓
  "Return to Hub" button

INTERTEXT / ECHO PAGE (final body page before sources)
  Synthesizes the OT pattern across Scripture
    ↓
  NT Connection Cards — one per major NT echo
    ↓
  Type → Anti-Type Grid — climactic synthesis
    ↓
  Study Complete Banner — rainbow gradient celebration that bookends the Rainbow Covenant Hero
```

When to use this recipe: any character or theme that needs ≥5 pages and benefits from spectrum-based wayfinding. When NOT to use: short studies (≤4 pages) where a single page with sections is plenty, or where the color spectrum carries no narrative work.

## Quick decisions

### Hero choice
- **Cosmic Halo** → flagship thematic studies, capstone pages
- **Herringbone Light** → Connections Corner pages
- **Animated Gradient** → about / hub / lay-audience pages
- **Braided Rope** → corner-level hubs that aggregate
- **Cosmic Spectrum** → tool pages, workbenches, generative resources (Sermon Builder, Practice Lab)
- **Rainbow Covenant** → multi-page hubs that color-code their child pages along a spectrum (Noah's 8-page hub)

### When to use the Chiasm component
✅ Mirror structure with a clear theological pivot  
✅ 5–11 layers (ABA, ABCBA, etc.)  
❌ Just any old structural outline — use a regular ordered list

### When to use Chiastic Structure vs. Chiastic Two-Column with Center Pivot
- **Chiastic Structure** (vertical indent ladder) = short chiasms (≤5 pairs) where the rhythm and indent depth carry the reading
- **Chiastic Two-Column with Center Pivot** = long chiasms (5+ pairs) where readers need help seeing what mirrors what; the side-by-side layout makes correspondence explicit

### When to use Pattern Chain vs. Expansion Track vs. Visual Timeline
- **Pattern Chain (Icon Trail)** = typology progression where icons carry meaning (Garden → Ark → Tabernacle → Temple → God) — lightweight, ≤7 nodes
- **Expansion Track** = sequential stages where the *increase in scope* is the point (one person → family → tribe → nation)
- **Visual Timeline** = events with chronology where the *time dimension* is the argument

### When to use Pivot Box vs. Key Insight vs. Core Logic Quote
- **Pivot Box** = single-verse centerpiece marking the literary *structural center* of a chiasm or narrative ("And God remembered Noah")
- **Key Insight** = reader-help sidebar; "here's what to notice"
- **Core Logic Quote** = thesis-statement quote that frames the page's argument

### When to use Genealogy Chain vs. Visual Timeline
- **Genealogy Chain (Numbered Pills)** = generations with names (10 generations Adam → Noah, 14 generations Babylon → Christ); star-modifier highlights named figures
- **Visual Timeline (State-Coded)** = events with dates or states; uses color to encode status

### When to use Methodology Card vs. Academic Note Box vs. Critical Distinction
- **Methodology Card** = describing how the study was *made* (literary-canonical reading, ANE comparison, intertextual method)
- **Academic Note Box** = hermeneutical *posture* — how disputed passages are handled, what level of certainty is claimed
- **Critical Distinction** = correcting a specific common *misreading* in the body of the study

### When to use the Atlas Node vs. Goat Panel
- **Atlas Node** = many parallels (3+) you want filterable
- **Goat Panel** = exactly two opposed concepts side-by-side

### When to use the Atlas Node vs. Passage Card
- **Atlas Node** = paired comparison (left passage / right passage / rationale) with bullet-list metadata
- **Passage Card** = single passage with prose commentary; has a meta block but the body is paragraph-form

### When to use which structural diagram
- **Chiasm** = mirror structure with a clear theological pivot, 5–11 layers
- **Mirror-+-Center Infographic** = mirror structure where you also need a *visible* center pivot box (Torah architecture, large-scale book-level chiasm)
- **Arc Diagram** = 30+ connections you want filterable and hover-explorable
- **Visual Timeline** = sequential stages where each has a *moral verdict* (success / failure / awaiting)
- **Trajectory Flowchart** = sequential stages with phases that fork or branch
- **Severity Spine** = stages encoding *intensity* (mild → severe), not state
- **Status Progression** = transformation between *categorically different states* (impure → pure → holy)
- **Expansion Track** = stages that are simply *more* of the same (one → some → many → all)
- **Paired Flow Tracks** = two parallel sequences in a 1:1 comparison (Lev pattern ⇄ Luke pattern)
- **ASCII Diagram Block** = the diagram itself is the artifact (screenshot, slide, share)

### When to use which 4-quadrant component
- **Four-Circle Venn (Flippable)** = each quadrant fits on a flip-back face (≤ ~30 words)
- **Portrait Venn (2×2 + Center)** = quadrants need paragraph-length prose; static
- **Concepts Grid** = 4 *categories* previewed at the top of a section (not a deep dive)

### When to use which top-of-page wayfinding
- **Mini-TOC Pill Bar** = horizontal section-jumps under the hero, utility weight
- **Quick Links Pills** = large rounded pills, primary affordance, hub-page weight
- **TOC Card** = mid-page or post-intro inline TOC, 6+ sections in 2 columns
- **Quick Nav Sidebar Dots** = ambient, desktop-only, floats; for very long studies

### When to use which "synthesis at the end" callout
- **Key Insight Box** = a one-sentence takeaway anywhere in the body
- **Core Logic Quote** = the page's central thesis in italic, central
- **Critical Distinction Box** = correcting a common reader error
- **Pattern Summary Box** = end-of-section synthesis stating the *structure*
- **One-Line Takeaway Box** = page-closer stating the *practical upshot*
- **Revolutionary Inversion Box** = the moment the pattern flips (mid-section, attention-grabbing)

### When to use the Hebrew Vocab Card vs. the Color-Coded Grid
- **Hebrew Vocab Card** = featured single term (4–8 per study, no system relationship)
- **Hebrew Vocab Color-Coded Grid** = closed lexical system where the terms *oppose each other* and the colors carry the structural argument (purity binaries, covenant types, sacrifice categories)

### When to use which output container
- **Output Code Block** = a generated artifact the user wants to copy out (HTML / XML / Markdown / plain text)
- **Copy Box (Template)** = a *blank scaffold* the user fills in elsewhere (team-prep template, prayer request, study outline)
- **Scripture Quote / Scroll Text** = body-content scripture passages (these belong in prose, not in a code panel)

### When to use which "browse" pattern
- **Nested Accordion Browser** = 2–3 nesting levels with state (Sermon Builder canon picker)
- **Tanakh Reference Table** = flat reference with the original-language column carrying weight
- **Page Navigation Grid** = study with multiple sub-pages, all equally weighted, no hierarchy
- **Featured Grid** = curated highlights, not exhaustive coverage

### When to use which sequential-step pattern
- **Multi-Step Wizard** = the user is *producing* something through ordered choices, and each step's output shapes the next
- **Pathway Navigation** = the user is *reading through* a curriculum in order; each page is content, not configuration
- **Visual Timeline** = stages of a *narrative* (Eden → Babel → Christ), not stages of a *user flow*

### When to use Validation Message vs. Critical Distinction
- **Validation Message Box** = UX feedback ("you missed a required field") — toggles on/off based on form state
- **Critical Distinction Box** = body content ("readers commonly misunderstand X to mean Y") — always visible, theological correction
- They look superficially similar (both red-tinted callouts) but serve different purposes; don't substitute one for the other.

### When to use Mini-Card Grid vs. plain Theology Card
- **Mini-Card Grid** = 4–6 short equal-weight concepts
- **Theology Card** = one concept with paragraph treatment

### When to use Flip Card vs. native &lt;details&gt; accordion
- **Flip Card** = surprise reveal earns the motion (parallel revelations, paired insights)
- **Accordion** = optional deeper info that not every reader needs

### When to use Iceberg
✅ Symptom / diagnosis / solution argument  
✅ Surface vs. depth metaphor  
❌ Just any old hierarchy — use a Trajectory Flowchart instead

## Worth Considering (Not Yet On The Site)

Each of these solves a problem the current pattern library doesn't have a clean answer for.

### High-value additions

**1. Verse Hover Cards (Sefaria-style)**  
Inline references like "Gen 15:6" become hover-cards showing the actual verse text. Closes the gap where readers either trust the citation or have to look it up. Implementation: small JSON file of common verses + hover JS. Best paired with your existing `.scripture-quote` styles.

**2. Footnote Hover-Cards**  
Inline superscript citation markers (`<sup>[3]</sup>`) that hover-pop a small card with the full bibliography entry. You already have rich `<details class="bibliography-section">` blocks; this surfaces them inline without scrolling.

**3. Audio Pronunciation Pill**  
Hebrew/Greek words with a small play button. Web Speech API or recorded MP3. Closes the accessibility gap for readers who know what `שָׁלוֹם` means but can't sound it out confidently.

### Medium-value additions

**4. Theme Heat Map Grid**  
GitHub-contribution-style grid where rows = books and columns = chapters; cell intensity = how often a theme appears. Dense second-view of your Genesis Connections / Atlas datasets.

**5. Dual-Pane Compare Strip**  
Inline left-passage / center-arrow / right-passage with shared markup highlighting. Stripped-down mid-paragraph version of the Atlas Node.

**6. View-Mode Toggle**  
Three-way toggle to switch a passage between reading view, Hebrew/Greek view, and markup view. Best for your LLT-SSE pages where the layered markup already exists — this just exposes it on demand.

### Lower-effort wins

**7. Section-Read Indicator (Mini-TOC dots)**  
A small dot next to each section in the Mini-TOC pill bar that fills in as the reader scrolls past that section — like a checklist of what they've covered. The Mini-TOC already exists with `aria-current="true"` styling on the active link; this would extend it with a `.read` state on past sections that fills the dot with `--page-accent`. Lower-effort than a full progress system because the IntersectionObserver wiring is already used by the qdot sidebar.

## Pattern Anti-Patterns to avoid

- **`animate-on-scroll` fade-ups that hide content until JS reveals it.** They delay the reader's access to information; they don't carry information. Use motion only when the motion *is* the meaning (flip = reveal, hover = preview, expand = depth).
- **Stacking 3+ callouts in a row.** Key Insight + Critical Distinction + Core Logic + Workflow Thread together creates emphasis-fatigue. Pick one per section.
- **Inventing ad-hoc hex codes for accents.** All color must come from the six `:root` accent tokens (`--accent-purple`, `--accent-rose`, `--accent-blue`, `--accent-magenta`, `--accent-indigo`, `--accent-teal`) or `color-mix()` blends of them. Custom hex breaks the page-accent system in multi-page suites.
- **More than 3 fonts on a page.** The current site uses Cormorant Garamond / DM Sans / JetBrains Mono — that's the budget.
- **Replacing the Theology Card with a Mini-Card Grid for a single concept.** The grid implies "these are equal-weight peers." If they're not, use one Theology Card with structured headings inside.

> **Note:** Earlier versions of this catalog flagged decorative gradients on body content as an anti-pattern. **v3.1 reverses that call** — the purple→rose gradient on tags, badges, indicators, timeline rails, progress bars, and active-state pills is the v2 saturated signature look the site was built on, and it's the *correct* approach in v3.1. Gradients are still inappropriate for body *prose* and *card backgrounds*, but on small interactive components they are the house style.

### Palette and tokens (v3.1)

All accents come from `:root` in `global-v3.css`. The six accent tokens are:

| Token | Hex | Role |
|---|---|---|
| `--accent-purple` | `#7209b7` | Page default, structural depth, primary buttons |
| `--accent-rose` | `#e11d48` | Critical Distinction, gradient pivot |
| `--accent-blue` | `#00b4d8` | Information, horizon, optional indicator gradient |
| `--accent-magenta` | `#ec4899` | Pivot Box glow, song-link warm gradient |
| `--accent-indigo` | `#4f46e5` | Word Study, depth bridge |
| `--accent-teal` | `#14b8a6` | Pattern Summary, success states |

Five named gradients are pre-built: `--gradient-primary` (purple→rose, the v2 signature, used on tags / badges / progress / timeline / qdot / mini-toc / cite-button / cosmic-button / character-type-badge / enhancement-indicator), `--gradient-cool` (blue→indigo), `--gradient-warm` (rose→magenta, used on song-link), `--gradient-fresh` (teal→blue, used on cite-button.copied), `--gradient-deep` (indigo→purple).

Legacy v2/catalog tokens (`--ink`, `--paper`, `--rule`, `--gold`, `--plum`, `--crimson`, `--olive`) still resolve as aliases pointing at the v3.1 palette, so existing pages render without modification.
